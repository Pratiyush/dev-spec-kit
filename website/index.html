The moat

Verification by construction

Three interlocking properties that make "done" mean something.

🔏

Evidence-bound done

Every task is bound to one or more @check refs at creation time. dev-spec-kit task done is a hard gate — it reads the live proof graph and refuses if any bound check has not produced a passing run. Not a lint warning. Not a suggestion. An exit-code-1 refusal.

verify.blockDoneOnFail: true
🌊

Drift detection

Proofs stamp the tested tree-hash, not the commit SHA. When you change code after a green run, that proof goes stale automatically — the moat can't be gamed by committing already-tested code. dev-spec-kit drift finds stale proofs and re-runs their checks in one move. dev-spec-kit graph build exits 1 while any proof is red or stale.

memory.driftDetection: true
🛡️

Hard gate hooks

PreToolUse hooks run dev-spec-kit guard pr before every gh pr create call — blocking PR creation while any proof is not green. Missing graph also blocks (absence of state ≠ permission). An anti-tamper lock prevents the agent from editing specs or bound test files while a task is in flight — the moat cannot be edited by the thing it gates.

gates.facts: off → on for investigative gates
The loop

Spec → check → done — one proven step at a time

Watch a full verified change from spec to approved PR, with real command output at every gate.

dev-spec-kit workflow demo
# 1 · Route the request
$ dev-spec-kit route "add idle-session timeout after 30 min"
Mode: full-spec (reason: new behaviour, not a tiny fix)
Confirm? [y/n]: y
# 2 · Spec
$ cat .dev-spec-kit/specs/session.md
## Requirement R-SESSION-01 — idle timeout
WHEN a user session has been idle for >= 30 min THEN the system SHALL invalidate it.
@check kind=unit ref=SessionTest#idleTimeout
# 3 · Derive tasks
$ dev-spec-kit spec tasks
✓ T3 idle timeout [bound: SessionTest#idleTimeout]
$ dev-spec-kit task start T3
# 4 · TDD red phase
$ dev-spec-kit check run T3 SessionTest#idleTimeout --stack java-maven --expect-red
✗ FAILED (expected red — TDD phase) tree: a3b2c11 dirty: true
→ implement, then re-run without --expect-red
# 5 · Prove
$ dev-spec-kit check run T3 SessionTest#idleTimeout --stack java-maven
✓ SessionTest#idleTimeout PASSED tree: a7f3c91 dirty: false
proof recorded in journal (worst-of rollup: GREEN)
# 6 · Done gate
$ dev-spec-kit task done T3
✓ T3 done 🎉 (1/1 checks green)
T3 — idle timeout100%
# 7 · Build graph · Approve · PR
$ dev-spec-kit graph build
✓ graph green (1 requirement · 1 criterion · 1 proof all GREEN)
$ dev-spec-kit approve T3 --note "timing verified; logic correct"
🛡️ approval recorded → .dev-spec-kit/approvals/T3-2026-06-11.json
$ dev-spec-kit pr --title "feat: idle session timeout" --create
✓ guard pr — all proofs GREEN · creating PR…
https://github.com/org/repo/pull/42
$
Everything in the box

One tool, the full enforcement stack

Every surface uses the same proof state — no inconsistency between the CLI, the hook, and the board.

🟢

Evidence-bound tasks

Tasks carry @check bindings. Done is physically enforced, not trusted.

🔴

TDD red→green loop

Write the test first, run it with --expect-red, then implement. The runner records the full proof chain.

🟣

Gate packs

Security triggers floor the routing tier. Required spec sections enforced per pack. User-editable YAML, not code.

⚪

Unproven = blocked

Missing graph, stale proof, or unrun criterion — all treated identically by the guard and the PR gate.

📊

LEDGER + TRACKING boards

Auto-generated boards from the journal that cannot lie — no manually edited markdown.

🛡️

Anti-tamper lock

Specs and bound test files lock once green. Human escape hatch: dev-spec-kit unlock, time-boxed, journaled.

🔏

Tree-hash proofs

Proofs identify the exact tested tree, not a commit SHA. Code changes after a proof auto-stale it.

🌊

Wave parallelism

Group independent tasks, dispatch to git worktrees concurrently. Cap default 6 — avoids rate-limit wipeouts.

🕸️

graphify integration

20+ language code-graph. Drift, blast-radius, affected, and reuse queries run against the live graph.

📓

Journal audit trail

Every CLI invocation is a timestamped JSONL event. dev-spec-kit log replays the history with emoji lane.

🔁

Learning loop

Lessons from retros promote into laws or permanent regression tests. Bugs that can come back silently were never fixed.

⚡

Ceremony proportional to size

Small changes take the quick path. Full-spec pipeline only when a feature earns it. Routing is config-driven.
